1. Introduction

SkillForge is an AI-driven educational platform designed to enhance the learning experience through personalization and automation. The system provides tailored learning content and auto-generates exams based on learner performance, bridging the gap between static learning materials and dynamic student needs.

The purpose of this document is to define the Product Backlog and detailed features planned for development in Milestone 1. Adopting an Agile methodology, this document outlines the user stories, priorities, and specific modules required to build the core functionality of the system.

This backlog will serve as the primary reference for planning sprints, tracking development progress, and evaluating the successful completion of Milestone 1 tasks. It ensures that the development team maintains a clear focus on delivering high-value features efficiently.

3. Sprint-1 Scope (Milestone 1)

Sprint 1 focuses on delivering the Minimum Viable Product (MVP) for SkillForge. The objective is to establish the core architecture and enable the fundamental flow of the application: a user registering, an instructor creating a course/quiz, and a student attempting that quiz with immediate results.

Sprint 1 will include the following features:

  • User Registration & Authentication (Secure login for all roles)
  • Role-Based Access Control (Student vs. Instructor views)
  • Course & Topic Management (Basic structure creation)
  • Question Bank Management (Adding questions to database)
  • Basic AI-Based Exam / Quiz Generator (Logic for question selection)
  • Instant Evaluation & Feedback (Automated grading for MCQs)
  • Basic Performance Analytics (Score history)
Sprint Goal: Deliver a working end-to-end flow where a student can register, log in, enroll in a course, attempt an auto-generated quiz from the question bank, and view their score and basic feedback.

4. Conclusion

The Product Backlog defined in this document provides a clear, structured roadmap for the development of SkillForge. By prioritizing core functionalities such as the Exam Generator and Role-Based Access Control in Sprint 1, the team ensures that a functional prototype is ready for Milestone 1 evaluation.

Moving forward, this backlog will serve as a living document. It will be refined in future sprints as the team gathers feedback from the initial deployment and integrates more complex AI-driven adaptive capabilities.
